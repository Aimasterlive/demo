Introduction:

Git is a distributed version control system designed to manage and track changes in
source code during software development. Developed by Linus Torvalds, Git allows multiple developers
to work on a project simultaneously, maintaining a clear history of modifications. It emphasizes flexibility,
speed, and collaboration, making it an essential tool for individual developers and teams alike. Git is widely
used in the software development industry and is supported by various hosting platforms like GitHub and GitLab.

Follow Steps to Install Git:

Step-1: Update System Packages

Make sure your system is up-to-date by running the following commands

$ sudo yum update
$ sudo yum upgrade

Step-2: Install Git using the package manager

$ sudo yum install git

Step-3: Verify Git Installation

Check if Git is installed successfully by running command

$ git --version

Step-4: Configure Git

We need to configure Git with your name and email. Replace Your Name and your.email@example.com with your actual name and email:

$ git config --global user.name "Your Name"
$ git config --global user.email "your.email@example.com"

Adding Work Folder to Git Repository

Step-1: git init

Initializes a new Git repository in the current directory

$ git init

Step-2:git add

Stages changes for commit. This command tells Git that you want to include updates to a particular file in the next commit

$ git add "filename"

To stage all changes

$ git add .

Step-3: git commit

Records the changes staged in the previous step. It's like taking a snapshot of your changes

$ git commit -m "Your commit message here"

Step-4: git remote add origin

Adds a remote repository. The origin is a common name for the default remote repository

$ git remote add origin "repository_url"

Step-5:git push

Pushes your changes to a remote repository. The -u flag is used to set up tracking between the local branch and the remote branch

$ git push -u origin master
